编程机制

C语言实现通过编译和链接两个步骤来完成，编译器将源代码转换成中间代码，链接器把中间代码和其他代码合并，生成可执行文件。

![[Pasted image 20230807203648.png|400]]

源代码经过编译器，转换为目标代码；
启动代码充当着程序和操作系统之间的接口；

链接器将目标代码、系统的标准启动代码和库代码合并成一个文件，即可执行文件。

缓冲区

老式系统在用户输入字符后立即重复打印该字符，属于无缓冲（或直接）输入；
大部分系统在用户按下Enter键之前不会重复打印，属于缓冲输入。

用户输入的字符被收集并储存在一个被称为缓冲区（buffer）的临时存储区。
![[Pasted image 20230817170640.png|500]]

缓冲分为：完全缓冲I/O和行缓冲I/O

  • 完全缓冲：当缓冲区被填满时才刷新缓冲区，通常出现在文件输入中。缓冲区的大小取决于系统
  • 行缓冲：在出现换行符时刷新缓冲区。键盘输入通常是行缓冲输入

函数

函数是完成特定任务的独立程序代码单元

函数定义时给出的参数成为形式参数，简称形参；函数调用时给出的参数称为实际参数，简称实参。函数调用时，将实参的值传递给形参，相当于一次赋值操作。

注意：⚠

  • C语言不允许函数嵌套定义，即不能在一个函数中定义另一个函数，必须在所有函数之外定义另外一个函数。

形参和实参的区别和联系

  1. 形参只有在函数被调用时才会分配内存，调用结束后，立即释放内存；形参只在函数内部有效，不能在函数外部使用。
  2. 实参可以是参量、变量、表达式、函数等，进行函数调用时，必须提前赋值；
  3. 实参和形参在数量、类型、顺序上必须严格一致，否则会发生“类型不匹配”的错误；
  4. 函数调用中发生的数据传递时单向的，只能把实参的值传递给形参，形参的值发生改变不会影响实参。
  5. 形参和实参可以同名，相互独立，互不影响。

函数返回值

  • 没有返回值的函数为空类型，用void表示
  • return语句可以有多个，可以出现在函数体的任意位置；每次调用函数只能有一个return语句被执行，只有一个返回值；
  • 函数遇到return语句就立即返回，后面的语句不会被执行，return语句具有强制结束函数执行的作用；
  • return语句是提前结束函数的唯一办法

函数的嵌套调用

函数不能嵌套定义，可以嵌套调用

如果一个函数A()在定义或调用过程中出现对另一个函数B()的调用，则称A()为主调函数或主函数，称B()为被调函数。
当主调函数遇到被调函数时，主调函数会暂停，CPU转而执行被调函数；被调函数执行完毕再返回主调函数。

函数声明

函数声明给出了函数名、返回值类型、参数列表等与该函数有关的信息，称为函数原型（Function Prototype）
格式：dataType functionName(dataType1 param1, dataType2 param2...);

对于多文件的程序，通常将函数定义放在源文件（.c文件）中，将函数的声明放到头文件（.h文件）中，使用函数时引入对应的头文件即可。

全局变量和局部变量

  • 局部变量
    定义在函数内部的变量称为局部变量（Local Variable），作用域仅限于函数内部
    说明：
  1. main()函数中定义的变量也是局部变量，只能在main函数中使用，main函数与其他函数地位平等；
  2. 形参变量、函数体内定义的变量都是局部变量；
  3. 不同的函数中可以使用相同的变量名，表示不同的数据，分配不同的内存；
  4. 在语句块中也可以定义变量，作用域仅限于当前语句块。
  • 全局变量
    在所有函数外部定义的变量称为全局变量（Global Variable），作用域默认为整个程序，包括.c和.h文件。

函数内部的局部变量和函数外部的全局变量同名时，当前函数的局部作用域中，全局变量会被屏蔽，函数内部使用的是局部变量，而不是全局变量。

变量的使用遵循就近原则，如果在当前的局部作用域中找到同名变量，就不会再去更大的全局作用域中查找。

  • 代码块
    由{}包围起来的代码，如函数体、选择结构、循环结构等。
    C语言允许在代码块内部定义变量，变量具有块级作用域；代码块内部定义的变量只能在代码块内部使用

递归函数

函数调用自身称为递归调用，这种函数称为递归函数。

尾递归

把递归调用至于函数的末尾，即正好在return语句之前，这种形式的递归被称为尾递归（tail recursion）。尾递归是最简单的递归形式，相当于循环。

递归的优缺点

优点：为某些编程问题提供了最简单的解决方案
缺点：一些递归算法会快速消耗计算机的内存资源

main()函数是主函数，可以条用其他函数，而不允许被其他函数调用；
C程序的执行总是从main()函数开始，完成对其他函数的调用后再返回到main()函数，最后由main()函数结束整个程序。

数组和指针

指针（pointer）是一个值为内存地址的变量

  • 地址运算符：&（取址）
    表示变量的地址

  • 间接运算符:*（取值）
    后面跟一个指针名或地址时，给出储存在指针指向地址上的值

指针定义：声明指针变量时，必须指定指针所指变量的类型
不同的变量类型占用不同的存储空间，一些指针操作要求知道操作对象的大小；程序必须知道储存在指定地址上的数据类型。
int * pi;
char * pc;
float * pf, * pg;

*和指针名之间的空格可有可无，通常在声明时使用空格，在解引用变量时省略空格

数组

初始化

与普通变量类似，在使用数组元素之前，必须先赋初值
当初始化列表的值少于数组元素个数时，编译器会把剩余的元素都初始化为0；
如果初始化列表的项数多余数组元素个数，编译器会报错

指定初始化器

可以在初始化列表中使用带方括号的下标指明待初始化的元素

对于一般的初始化，在初始化一个元素后，未初始化的元素都会被设置为0

  • 特性
  1. 若指定初始化器后面还有更多值，则后面这些值将用于初始化指定元素后面的元素
  1. 再次初始化指定的元素，则最后的初始化将会取代之前的初始化

数组名是该数组首元素的地址

指针

地址按照字节编址，short类型占用2字节，doubel类型占用8字节
指针加1代表下一元素的地址，而不是下一个字节的地址。

指针变量的基本操作

  • 赋值：可以把地址赋给指针
  • 解引用：*运算符给出指针指向地址上储存的值
  • 取址：&运算符给出指针本身的地址
  • 指针与整数相加：整数与指针所指向类型的大小（以字节为单位）相乘，然后再把结果与初始地址相加
  • 递增地址：递增指向数组元素的指针可以躺该指针移动至数组的下一个元素
  • 指针求差：计算两个指针的差值
  • 比较：使用关系运算符可以比较两个指针的值，前提是两个指针都指向相同类型的对象。

字符串输入

先预留储存该字符串的空间，然后用输入函数获取该字符串

gets()函数

scanf()只能读取一个单词；gets()函数能够读取整行输入，直到遇到换行符，然后丢弃换行符，储存其余字符，并在这些字符的末尾添加一个空字符使其称为一个C字符串

注意：get()函数只知道数组的开始处，并不知道数组中有多少个元素，如果输入的字符串过长，会导致缓冲区溢出(buffer overflow)，即多余的字符超出了指定的目标空间。

fgets()函数&fputs()函数

fgets()函数通过第二个参数显示读入的字符数来解决溢出问题

若将fgets()函数的第二个参数设置为nnn，fgets()函数会读入n−1n-1n−1个字符，或者读到第一个换行符为止

fgets()函数将第3个参数指明要读入的文件，如果读入从键盘输入的数据，则以stdin作为参数

fputs()函数第2个参数指明要写入的文件，如果要显示在计算机显示器上，应使用stdout作为参数

scanf()函数

scanf()函数有两种方法确定输入结束，

  1. 从第一个非空白字符作为字符串的开始，以下一个空白字符作为字符串的结束
  2. 指定字段宽度，如%10s，scanf()将读取10个字符或读到第1个空白字符停止

用fgets()读取从键盘输入的数据更合适；scanf()的典型用法是读取并转换混合数据类型为某种标准形式。

字符串输出

puts函数

把字符串的地址作为参数即可；puts()在显示字符串时会自动在其末尾添加一个换行符
puts函数在遇到空字符时停止输出

fputs()函数

fputs()函数的第2个参数指明要写入数据的文件，

与puts()不同，fputs()不会再输出的末尾添加换行符

printf()函数

将字符串的地址作为参数；printf()不会自动在每个字符串末尾加上一个换行符

字符串函数

strlen()

strlen()函数用于统计字符串的长度

strcat()

用于拼接字符串，函数接受两个字符串作为参数，把第2个字符串的备份附加在第1个字符串末尾，并把拼接后形成的新字符串作为第1个字符串，第2个字符串不变

strncat()

strcat()函数无法检查第1个数组是否可以容纳第2个字符串，如果分配给第1个数组的空间不够大，多出来的字符溢出到相邻储存单元时会出现问题。
解决方法：

  1. 使用strlen函数查看第1个数组的长度；
  2. 使用strncat函数，该函数的第3个参数指定最大添加字符数；

strcmp()

字符串比较函数，该函数通过比较运算符来比较字符串，如果两个字符串参数相同，返回0，否则返回非零值。

strcmp()函数可以比较储存在不同大小数组中的字符串

strcpy()

strcpy()函数用于拷贝整个字符串

strcpy()函数的属性：

  1. strcpy()函数的返回类型是char *，该函数返回的第1个参数值，即一个字符的地址
  2. 第1个参数不必指向数组的开始

存储类别、链接和内存管理

  • 作用域：描述 程序中可访问标识符的区域
    一个C变量的作用域可以是块作用域、函数作用域、函数原型作用域或文件作用域

  • 链接
    C变量有3种链接属性：外部链接、内部链接或无链接

  • 存储期
    C对象有4种存储期：静态存储期、线程存储期、自动存储期、动态分配存储期

静态存储期的对象在程序的执行期间一直存在；
线程存储期用于并发程序设计，程序执行可被分为多个线程，具有线程存储期的对象，从被声明时到线程结束一直存在。
自动存储期：块作用域的变量通常都具有自动存储期，当程序进入定义这些变量的块时，为这些变量分配内存；当退出这个块时，释放刚才为变量分配的内存。

分配内存：malloc()和free()

malloc()函数和free()函数的原型在stdlib.h头文件中

malloc()函数

接受一个参数，所需的内存字节数；malloc()函数会找到合适的空闲内存块；

malloc()函数声明：void *malloc(size_t size)

返回一个指针，指向已分配大小的内存；如果请求失败，则返回NULL。

注意事项：

  • malloc()函数返回的是无类型指针，在使用时一定要强制转换为所需要的类型。
  • 使用完malloc开辟的空间后，一定要释放空间，否则会造成内存泄漏。
  • 使用malloc()开辟的空间时，不能进行指针的移动，否则会导致申请的空间和释放的空间大小不匹配。

free()函数

free()函数的参数是malloc()返回的地址，释放之前malloc()分配的内存

🐖注意：
使用后该指针变量一定要重新指向NULL，防止悬空指针（失效指针）出现，有效规避错误操作。

  • calloc()函数
    返回指向char的指针，接受两个无符号正数作为参数，第1个参数是所需的存储单元数量，第2个参数是存储单元的大小（以字节为单位）

文件

文件是在磁盘或固态硬盘上的一段已命名的存储区，所有文件的内容都以二进制形式储存。

为了规范文本文件的处理，C提供两种访问文件的途径：二进制模式和文本模式。
二进制模式中，程序可以访问文件的每个字节；文本模式中，程序所见的内容和文件的实际内容不同。

打开文件

操作文件之前必须先打开文件，即让程序和文件建立连接的过程。

标准输入文件stdin(表示键盘)、标准输出文件stdout（表示显示器）、标准错误文件stderr（表示显示器）

fopen()函数

使用fopen()函数打开文件，该函数声明在stdio.h中。

第一个参数是待打开文件的名称，更确切地说是一个包含该文件名的字符串地址；
第二个参数是一个字符串，指定待打开文件的模式。

打开文件后，fopen()将返回文件指针，其他I/O函数可以使用这个指针指定该文件。

FILE *fp = fopen("demo.txt","r")
表示以”只读“的方式打开当前目录下的demo.txt文件，并使fp指向该文件，即可通过fp来操作demo.txt。fp通常被称为文件指针。

文件指针的类型是指向FILE的指针，FILE是一个定义在stdio.h中的派生类型。
文件指针fp并不指向实际的文件，它指向一个包含文件信息的数据对象，其中包含操作文件的I/O函数所用的缓冲区信息。

  • 判断文件是否打开成功
    通过判断fopen()返回值来判断文件是否打开失败

整体来说，文件打开方式由r、w、a、t、b、+六个字符拼成，各字符的含义是：
r（read）读；w（write）写；a（append）追加；
t（text）文本文件；b（binary）二进制文件；+读和写

getc()和putc()函数

ch = getc(fp) 表示从fp指定的文件中获取一个字符；
putc(ch,fpout)表示把字符ch放入FILE指针fpout指定的文件中

fgets()和fputs()函数

  • fgets()——用于从指定的文件中读取一个字符串，并保存到字符数组中
    fgets(char * str,int n,FILE *fp);
    第一个参数表示储存输入位置的地址（char * 类型）；第二个参数是一个整数，表示待输入字符串的大小；
    最后一个参数是文件指针，指定待读取的文件。
    读取成功返回字符数组首地址，即str；读取失败，返回NULL

fgets()函数读取输入直到第一个换行符，或者读到文件结尾，或者读取n-1个字符，然后fgets()在末尾添加一个空字符使之成为一个字符串。字符串大小是其字符数加上一个空字符

  • fputs()
    int fputs(char * str，FILE *fp);
    fputs()函数接收两个参数：第一个是字符串的地址；第二个是文件指针。
    该函数根据传入地址找到的字符串写入指定的文件中，写入成功返回非负数，失败返回EOF。

🔺fputs()在打印字符串时不会在末尾添加换行符（fgets()保留了换行符）

fread()和fwrite()函数

  • fread()函数——用来从指定文件中读取块数据，即若干个字节的数据，可以是一个字符、字符串，也可以是多行数据。
    size_t fread(void *ptr, size_t size,size_t count,FILE *fp);

  • fwrite()函数——用来向文件中写入块数据
    size_t fwrite(void * ptr, size_t size, size_t count, FILE *fp);
    参数说明：
    ptr为内存区块的指针，可以是数组、变量、结构体等；size表示每个数据块的字节数；count表示要读写的数据块的块数；fp表示文件指针

fprintf()和fscanf()函数

相比于printf()和scanf()，fprintf()和fscanf()需要用第一个参数指定待处理的文件。

rewind()函数让程序回到文件开始处，方便while循环打印整个文件的内容。
rewind()函数接收一个文件指针作为参数

fseek()和ftell()函数

  • fseek()——用于将位置指针移动到任意位置
    int fseek(FILE *fp, long offset, int origin);
    第一个参数是FILE指针，指向待查找的文件；
    第二个参数是偏移量（offset），该参数表示从起始点开始要移动的距离。必须是一个long类型的值，可以是正（前移）、负（后移）或0（保持不动）；
    第三个参数是模式，该参数确定起始点。

SEEK_SET —— 文件开始处
SEEK_CUR —— 当前位置
SEEK_END —— 文件末尾

文件结尾

getc()函数在读取到文件结尾时，将返回一个特殊值EOF；为了避免读到空文件，应使用入口条件循环进行文件输入。

关闭文件

文件一旦使用完毕，应该用fclose函数把文件关闭，以释放相关资源，避免数据丢失。

fclose()函数

fclose(fp)函数关闭fp指定的文件，必要时刷新缓冲区。对于较正式的程序，应该检查是否成功关闭文件，如果成功关闭，fclose()函数返回0，否则返回EOF。

位操作

原码、反码、补码

原码：
最简单的机器数表示法，用最高位表示符号位，其他位存放该数的二进制的绝对值。
以带符号位的四位二进制数为例：1010，最高位为1，表示一个负数，其他三位010，所以1010表示十进制-2

0001+0010=0011，1+2=3；
0000+1000=1000，+0+(-0)=-0；
0001 +1001=1010，1+(-1)=-2。

特点：

  • 原码直观、易懂，与真值转换容易；
  • 原码中0有两种不同的表示形式，使用不便；
    通常0的原码用+0表示，若计算过程中出现了-0，则需要用硬件将-0变成+0。
  • 原码表示加减运算复杂。

反码：
正数的反码等于原码；负数的反码等于原码除符号位外，按位取反。
以带符号位的四位二进制数为例：3是正数，反码与原码相同，可以表示位0011；-3的原码是1011，符号位保持不变，低三位按位取反，-3的反码位1100。

特点：

  • 反码表示中，用符号位表示数值的正负，形式与原码表示相同；
  • 反码中，数值0有两种表示方法；
  • 反码的表示范围与原码的表示范围相同。

补码：
正数的补码等于它的原码；负数的补码等于反码+1

  • 时钟计数法
    假设当前时间是3点，向前拨2个小时，向前拨14个小时，向后拨10个小时的结果一样，即：
    3:00 + 2h = 5:00
    3:00 + 14h = 5:00
    3:00 - 10h = 5:00

在时钟表示法中，2、14、-10都是一个相同数的不同表示形式，它们除以12，得到的余数都是2，即同余。
可以认为时钟计数法的模为12。

链表——链式存储结构

使用链表存储的数据元素，其物理存储位置是随机的。每个数据元素在存储时都配备了一个指针，用于指向自己的直接后继元素，

数组是顺序储存方式的数据结构
缺点：

  • 插入、删除操作所需要移动的元素很多，浪费算力；
  • 必须为数组设置足够的空间，否则有溢出风险。

链表的结点

链表中的每个数据的存储由以下两部分组成：

  1. 数据元素本身，其所在的区域称为数据域；
  2. 指向直接后继元素的指针，所在的区域称为指针域。

一个完整的链表由以下几部分构成：

  1. 头指针：一个普通指针，永远指向链表第一个节点的位置，用于指明链表的位置，便于后期找到链表并使用表中的数据；
  2. 节点：链表中的节点可以细分为头节点、首元节点和其他节点
  • 头节点：不储存任何数据的空节点，通常作为链表的第一个节点。（非必须）
  • 首元节点：链表中第一个存有数据的节点称为首元节点。首元节点只是对链表中第一个存有数据节点的称谓，无实际意义；
  • 其他节点：链表中剩余的节点；
